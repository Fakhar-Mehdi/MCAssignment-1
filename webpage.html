How to Start Using Git
Git commands explained

echo "message >> README.md:

Creates a README file in the current folder with the name mentioned after >> and stores the message written in double quotes in the README.md file

git init:

initiates a local git repository in your system

git add filename

Add the file with the mentioned filename in the local git repository

git add .

Add all the files in the directory to the local git repository

git config --global user.email "email-address":
git config --global user.name "name":

It is important to configure your Git username and email address as every Git commit will use this information to identify you as the author. You'll need to do this only once as you are using the --global option. It tells Git to always use this information for anything you do on that system.

git commit -m "message"

commits the files which are added by git add command. It stores all the commits along with the messages specified with them

git status"

Tells the current status of local repository. Itentifies new files and modifications as well.

git clone [url]

It creates a clone of repository which resides at the mentioned url

git diff

This command shows the file differences which are not yet staged.

git log

This command is used to list the version history for the current branch.

git branch

This command lists all the local branches in the current repository.

git checkout [branch name]

This command is used to switch from one branch to another.

git merge [branch name]

This command merges the specified branchâ€™s history into the current branch.

git push [variable name] [branch]

This command sends the branch commits to your remote repository.

git pull [Repository Link]

This command fetches and merges changes on the remote server to your working directory.
